# 社工的一日：今天也要接住誰？

一款關於第一線社工日常的網頁小遊戲，設計目的是讓一般民眾體驗社工在有限時間與資源中如何做取捨，並理解過勞是制度問題而非個人失敗。

---

## 遊戲目的

讓玩家在 90 秒內體驗社工面對的日常壓力：

- 同時處理多種性質不同的工作需求（危機、紀錄、會議、服務）
- 管理自身能量，避免過勞
- 在個案安全、紀錄壓力、服務進度之間做取捨
- 理解休息、督導、同事支持是實務工作的一部分，而非偷懶

---

## 操作方式

| 輸入 | 動作 |
|------|------|
| 方向鍵 / WASD | 移動社工角色 |
| 靠近任務卡 | 自動收集 |
| 靠近正確工作區 | 自動交付任務 |
| 手機左下角四向鍵 | 觸控移動 |

### 任務顏色對應區域

| 顏色 | 任務類型 | 正確工作區 |
|------|----------|------------|
| 🔴 紅色 | 危機事件 | 危機處理區 |
| 🔵 藍色 | 服務需求 | 個案服務區 |
| 🟡 黃色 | 行政紀錄 | 辦公桌 |
| 🟣 紫色 | 網絡聯繫 | 會議室 |
| 🟢 綠色 | 支持資源 | 督導室 或 休息角落 |

---

## 狀態指標

- **個案安全**：危機任務逾時會下降，完成危機任務會上升
- **社工能量**：隨時間自然下降，攜帶越多任務下降越快；完成綠色支持任務可恢復
- **紀錄壓力**：行政任務逾時會上升，完成後下降
- **服務進度**：完成藍色/紫色/紅色任務會上升

---

## 結局說明

| 結局 | 條件 |
|------|------|
| 撐住今天 | 個案安全 ≥ 60、社工能量 ≥ 30、紀錄壓力 < 80 |
| 勉強收尾 | 未達到「撐住今天」，但未系統過載 |
| 系統過載 | 個案安全 ≤ 0、社工能量 ≤ 0、或紀錄壓力 ≥ 100 |

## 如何部署到 GitHub Pages

1. 建立 GitHub 儲存庫（公開）
2. 將所有檔案推送到 `main` 分支：

```bash
git init
git add .
git commit -m "初始版本"
git remote add origin https://github.com/你的帳號/sw-day-game.git
git push -u origin main
```

3. 在 GitHub 儲存庫頁面進入 **Settings → Pages**
4. 來源選擇 `main` 分支，目錄選 `/ (root)`
5. 儲存後等待幾分鐘，即可在 `https://你的帳號.github.io/sw-day-game/` 玩到遊戲

## 倫理聲明

本遊戲設計遵循以下原則：
- 任務卡代表工作需求，不代表個案本人
- 不使用責備語言或污名化表達
- 休息與督導被設計為正向策略
- 遊戲結局強調制度壓力，而非個人失敗
