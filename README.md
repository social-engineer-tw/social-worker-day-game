# 社工的一日：一個都不漏接？

純前端公益倡議小遊戲。玩家在有限時間內接任務、送任務、使用休息室與功德無量，盡量達成本關 KPI，同時避免生命值歸零或壓力爆表。

## 遊戲流程

首頁主視覺 → 規則說明 → 第一關 → 關卡結果 / 升級 → 第二關

首頁按鈕不會直接開始遊戲，會先進入規則說明頁。玩家點擊「我知道了，開始第一關」後，才會正式開始第一關。

## 主視覺圖片

首頁主視覺圖片路徑：

```text
assets/hero-social-worker-day.png
```

若圖片尚未放入，首頁會顯示 fallback 背景與提示文字，不會讓頁面壞掉。

部署到 GitHub Pages 時，請確認 `assets` 資料夾有一起上傳。

## 操作方式

- 方向鍵 / WASD：移動社工角色
- 手機：使用畫面上的方向鍵
- Q / Tab：切換鎖定的手上任務
- 進入 ☕ 休息室：自動休息
- DEBUG：顯示或隱藏偵錯資訊

## 關卡

### 第一關：先撐住今天

- 倒數 60 秒
- KPI：完成 15 件任務
- 達成 KPI 會立即通關，不必等倒數結束

### 第二關：事情開始追上來

- 倒數 60 秒
- KPI：完成 18 件任務
- 任務生成略快
- 危機、文件、網絡任務比例略高

## 功德無量

每關倒數到 40 秒與 20 秒時，各生成一次 🙏 功德無量。

玩家碰到後會清空地圖上尚未接起的任務，但保留手上任務，不影響 KPI、不扣分、不重設倒數。

## 升級

每次關卡通過或 Game Over 後，都會抽出 3 張升級卡。玩家可選 1 張，下一關或下一局生效。

升級會保存在 `localStorage`；若瀏覽器限制儲存，仍會在本次開啟期間生效。

## 本機測試

直接用瀏覽器開啟 `index.html` 即可遊玩。不需要 npm、伺服器或資料庫。

## 技術限制

- HTML + CSS + 原生 JavaScript
- 不使用 React
- 不使用 npm
- 不使用 canvas
- 不使用後端
- 不使用外部圖片連結
